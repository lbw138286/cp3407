# Iteration 1 User Stories

Iteration 1 was selected from the highest priority user stories in the Week 2 product backlog. The priority 10 stories provide the basic foundation for the Smart Expense Tracker Web Application.

## Selected User Stories

| ID | Title | Description | Priority | Estimated Effort | Iteration 1 Status |
|---|---|---|---:|---|---|
| US1 | User Registration | As a user, I want to create an account so that I can securely store my financial data. | 10 | 3 Days | Todo |
| US2 | User Login | As a user, I want to log into the system so that I can access my expense records securely. | 10 | 2 Days | Todo |
| US3 | Add Expenses | As a user, I want to add daily expenses so that I can track my spending habits. | 10 | 4 Days | Done |
| US13 | Secure Password Protection | As a user, I want my account password to be securely protected so that my financial information remains safe. | 10 | 3 Days | Todo |
| US14 | Secure Database Storage | As a user, I want my financial data to be stored securely so that unauthorized users cannot access it. | 10 | 4 Days | Todo |

## Implementation Notes

US3 was selected for first implementation because expense entry is one of the core functional requirements of the system. Without this function, later features such as categorisation, reports, budget alerts, and trend analysis cannot work properly.
