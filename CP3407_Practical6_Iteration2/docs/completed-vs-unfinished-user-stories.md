# Practical 6: Completed vs Unfinished User Stories

Project: Smart Expense Tracker Web Application

## Completed User Stories

| User Story | Description | Iteration Completed | Evidence |
|---|---|---|---|
| US3: Add Expenses | User can add expense title, amount, category, and date. The record is displayed in a table and saved in local storage. | Iteration 1 | Implemented in `src/index.html`, `src/style.css`, and `src/script.js`. |
| US4: Edit and Delete Expenses | User can edit existing expense records and delete records that are no longer needed. | Iteration 1 | Implemented in `src/script.js` with edit, update, delete, save, and render functions. |

## Unfinished User Stories

| User Story | Description | Current Status | Planned Iteration | Reason Not Completed Yet |
|---|---|---|---|---|
| US5: Categorize Expenses | Improve category-based organization and filtering. | In Progress | Iteration 2 | The basic category field exists, but category filtering and review still need work. |
| US6: Set Monthly Budgets | Allow user to set a monthly budget limit. | Todo | Iteration 2 | Requires budget input, storage, and monthly spending calculation. |
| US7: Receive Budget Alerts | Alert users when expenses approach or exceed the budget. | Todo | Iteration 3 | Depends on US6 being completed first. |
| US8: Generate Monthly Financial Reports | Generate monthly spending summaries. | Future | Iteration 3 | Depends on categorized and stored expense data. |
| US9: Analyze Spending Trends | Show spending trends over time. | Future | Later iteration | Requires more expense history and reporting logic. |
| US10: Secure Account Login | Allow secure user login. | Future | Later iteration | Larger scope and should be planned separately. |

## Summary

Iteration 1 successfully completed the core expense record management features. Iteration 2 will continue from this foundation by improving categorization and adding monthly budget tracking.
