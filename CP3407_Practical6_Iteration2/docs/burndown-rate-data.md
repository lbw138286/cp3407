# Practical 6: Iteration 1 Burn Down Rate Data

Project: Smart Expense Tracker Web Application  
Graph: `burndown-rate-graph-iteration1.png`

## Purpose

The burn down rate graph shows the amount of remaining work during Iteration 1. It helps compare planned progress with actual progress.

## Data Used

| Days Left | Ideal Work Left | Actual Work Left | Explanation |
|---:|---:|---:|---|
| 4 | 4 | 4 | Iteration started with US3 and US4 planned. |
| 3 | 3 | 3 | Add Expense form and input capture work started. |
| 2 | 2 | 2 | Add Expense feature was completed and Edit/Delete work continued. |
| 1 | 1 | 0.5 | Most Edit/Delete tasks were completed, only final testing remained. |
| 0 | 0 | 0 | US3 and US4 were completed by the end of the iteration. |

## Interpretation

The actual line follows the planned line closely and reaches zero by the end of the iteration. This shows that Iteration 1 was completed successfully and the original plan was realistic.
