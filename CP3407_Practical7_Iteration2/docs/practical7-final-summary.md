# Practical 7 Final Summary

For Practical 7, the project applied Test-Driven Development to the Smart Expense Tracker Web Application. Five user stories were selected: US3 Add Expenses, US4 Edit and Delete Expenses, US5 Categorize Expenses, US6 Set Monthly Budgets, and US7 Receive Budget Alerts. For each user story, three test cases were written, giving a total of 15 planned test cases.

The project also includes 15 automated tests using JavaScript and the Node.js built-in test runner. The tests check both successful behaviour and error handling. The tested functions include adding expenses, validating expense input, editing expenses, deleting expenses, filtering by category, summarising category spending, setting budgets, calculating monthly spending, and returning budget alert statuses.

The testing work improves confidence in the core business logic of the project and provides a stronger foundation for Iteration 2 development.
