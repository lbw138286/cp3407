# Milestone 1 Iteration Plan

## Project
Smart Expense Tracker Web Application

## Planning Method
User stories are allocated into three iterations according to functional priority and estimated days of work. Iteration 1 focuses on the minimum working foundation of the system. Iteration 2 extends the system with budgeting and categorisation. Iteration 3 focuses on reports, analysis, and administration.

## User Story Allocation

| ID | User Story | Priority | Estimate | Iteration | Reason |
|---|---|---:|---:|---|---|
| US1 | As a user, I want to create an account so that I can securely store my financial data. | 10 | 3 days | Iteration 1 | Account creation is a foundation for secure personal expense records. |
| US2 | As a user, I want to log into the system so that I can access my expense records securely. | 10 | 2 days | Iteration 1 | Login is required before protected user data can be accessed. |
| US3 | As a user, I want to add daily expenses so that I can track my spending habits. | 10 | 4 days | Iteration 1 | Expense entry is the core transaction function of the application. |
| US4 | As a user, I want to edit and delete expenses so that I can correct mistakes in my records. | 10 | 3 days | Iteration 1 | Record maintenance is required immediately after expense entry. |
| US5 | As a user, I want to categorize expenses so that I can understand where my money is spent. | 9 | 3 days | Iteration 2 | Categorisation supports budgeting and reporting. |
| US6 | As a user, I want to set a monthly budget so that I can control my spending. | 9 | 3 days | Iteration 2 | Budget setup enables spending control. |
| US7 | As a user, I want to receive budget alerts so that I can avoid overspending. | 8 | 3 days | Iteration 2 | Alerts depend on expenses and budget data. |
| US8 | As a user, I want to generate monthly financial reports so that I can review my spending. | 8 | 4 days | Iteration 3 | Reports need accumulated expense data. |
| US9 | As a user, I want to view spending charts so that I can understand my financial behaviour visually. | 7 | 3 days | Iteration 3 | Charts are useful after reporting data exists. |
| US10 | As a user, I want to analyse spending trends so that I can improve future budgeting decisions. | 7 | 4 days | Iteration 3 | Trend analysis requires historical data. |
| US11 | As an admin, I want to manage user accounts so that system users can be controlled. | 6 | 3 days | Iteration 3 | Admin control is important but not required for the first user-facing increment. |

## Iteration Summary

| Iteration | Main Focus | Included User Stories | Total Estimate |
|---|---|---|---:|
| Iteration 1 | Core data entry and record management | US1, US2, US3, US4 | 12 days |
| Iteration 2 | Budgeting and categorisation | US5, US6, US7 | 9 days |
| Iteration 3 | Reports, analysis, and administration | US8, US9, US10, US11 | 14 days |
