# CP3407 Practical 8: Iteration 3

## Objective
Practice TEST-driven development for the Smart Expense Tracker Web Application.

## What is included

This package completes the Practical 8 requirements:

1. Uses UI designs and user stories as test specifications for the Iteration 3 TDD process.
2. Reflects on Iteration 2 and adjusts the plan for Iteration 3.
3. Calculates the actual velocity of Iteration 2.
4. Provides a burn down rate graph for Iteration 2.
5. Uses Iteration 2 velocity to update the backlog for Iteration 3.
6. Tracks Iteration 3 tasks and user stories with Todo, In Progress, and Done labels.
7. Updates GitHub Pages content for the completed Iteration 2 user stories.
8. Includes mock object framework research and a mock login implementation.
9. Includes executable automated tests for Iteration 3 features.

## Key Iteration 3 user stories

- US7: Receive Budget Alerts
- US8: Generate Monthly Financial Reports
- US9: Analyze Spending Trends
- US10: Secure Account Login with Mock Object Testing

## Velocity assumption

The team has 3 members and each member has 4 working days available.

Team capacity = 3 team members × 4 working days = 12 person-days

Iteration 2 completed work:

- US5: Categorize Expenses = 6 person-days
- US6: Set Monthly Budgets = 6 person-days

Actual velocity of Iteration 2 = 12 person-days
