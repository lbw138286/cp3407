# Iteration 2 Reflection

## Completed work

Iteration 2 focused on building the next layer of the Smart Expense Tracker Web Application after the basic add, edit, and delete expense functions were completed in Iteration 1.

The completed user stories were:

| User Story | Description | Status | Estimate |
|---|---|---:|---:|
| US5 | Categorize Expenses | Done | 6 person-days |
| US6 | Set Monthly Budgets | Done | 6 person-days |

## Reflection

Iteration 2 was completed successfully because the planned workload matched the team capacity. The team has three members, and each member had one working day per week across four working days, giving a total capacity of 12 person-days. The completed work also totalled 12 person-days, so the planning was realistic.

US5 was important because users need expense categories before they can understand their spending patterns. US6 was also important because budget alerts and reports depend on a monthly budget value. These two user stories provide the foundation for Iteration 3 features such as budget alerts, monthly reports, and trend analysis.

## What should be improved

The main improvement for Iteration 3 is to increase the use of test-driven development. Instead of only implementing features first, the team should use user stories and UI expectations as test specifications before and during implementation. Mock objects should also be introduced for login testing so that authentication logic can be tested without relying on a real server or database.
