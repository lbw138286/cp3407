# Iteration 3 Backlog Update

## Basis for planning

The actual velocity of Iteration 2 was 12 person-days. Therefore, the planned workload for Iteration 3 should also be close to 12 person-days.

## Selected user stories for Iteration 3

| User Story | Description | Estimate | Reason |
|---|---|---:|---|
| US7 | Receive Budget Alerts | 4 person-days | Depends on US6 monthly budget and helps users avoid overspending. |
| US8 | Generate Monthly Financial Reports | 4 person-days | Uses expense and category data to provide useful financial summaries. |
| US9 | Analyze Spending Trends | 2 person-days | Uses monthly totals to identify whether spending is increasing, decreasing, or stable. |
| US10 | Secure Account Login with Mock Object Testing | 2 person-days | Supports secure access and allows mock login testing without a real authentication server. |

## Total planned workload

```text
US7 + US8 + US9 + US10 = 4 + 4 + 2 + 2 = 12 person-days
```

## Deferred work

Additional reporting polish, database integration, and advanced admin account management are deferred because they exceed the Iteration 3 capacity.
